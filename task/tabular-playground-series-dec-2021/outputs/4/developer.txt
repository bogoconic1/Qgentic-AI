TypeError: XGBClassifier.fit() got an unexpected keyword argument 'early_stopping_rounds'

The error means you’re using a recent XGBoost where early_stopping_rounds is no longer accepted in fit(). Since v1.6.0 it must be set on the estimator (or via a callback), not passed to fit.

What to change
- Old (will raise your error on >=1.6):
  clf = xgb.XGBClassifier(**params)
  clf.fit(X_tr, y_tr, eval_set=[(X_val, y_val)], early_stopping_rounds=50, verbose=False)

- Fixed (set on the estimator):
  clf = xgb.XGBClassifier(**params, early_stopping_rounds=50)
  clf.fit(X_tr, y_tr, eval_set=[(X_val, y_val)], verbose=False)

- Or using the callback API:
  from xgboost.callback import EarlyStopping
  clf = xgb.XGBClassifier(**params)
  clf.fit(
      X_tr, y_tr,
      eval_set=[(X_val, y_val)],
      callbacks=[EarlyStopping(rounds=50, save_best=True)],
      verbose=False
  )

Notes
- You must pass eval_set when using early stopping.
- Don’t specify early_stopping_rounds in both constructor and callbacks/fit at the same time.
- If you prefer to keep your old code unchanged, pin XGBoost to <=1.5:
  pip install "xgboost<=1.5.2"

Why this happens
- In XGBoost 1.6.0+, early_stopping_rounds moved from fit() to the estimator to better match scikit-learn. See the 1.6 docs: “This parameter replaces early_stopping_rounds in fit() method.” ([xgboost.readthedocs.io](https://xgboost.readthedocs.io/en/release_1.6.0/python/python_api.html?utm_source=openai))
- Older versions (e.g., 1.2–1.5) did accept early_stopping_rounds in fit(), which is why older tutorials show that form. ([xgboost.readthedocs.io](https://xgboost.readthedocs.io/en/release_1.2.0/python/python_api.html?utm_source=openai))

TypeError: XGBClassifier.fit() got an unexpected keyword argument 'callbacks'

You’re passing callbacks to XGBClassifier.fit, but the xgboost version in your environment is too old for that API. Older releases of the scikit-learn wrapper didn’t accept callbacks, so Python raises “unexpected keyword argument 'callbacks'.”

Fix options (pick one):

1) Upgrade xgboost (recommended)
- Pip: pip install -U "xgboost>=1.7.6"
- Conda: conda install -c conda-forge "xgboost>=1.7.6"
- In a notebook: 
  - first cell: !pip install -q "xgboost>=1.7.6"
  - restart the kernel/session
After upgrading, your existing clf.fit(..., callbacks=[...]) will work.

2) Keep your current xgboost and remove callbacks
- Replace callbacks (e.g., EarlyStopping) with early_stopping_rounds and eval_set:

    clf.fit(
        X_train, y_train,
        eval_set=[(X_valid, y_valid)],
        eval_metric="logloss",     # or "mlogloss" for multiclass
        early_stopping_rounds=50,
        verbose=True
    )

3) Make the code version-agnostic (works on both old and new xgboost)
- Detect whether fit supports callbacks, and fall back to early_stopping_rounds:

    import inspect, xgboost as xgb
    clf = xgb.XGBClassifier(..., eval_metric="logloss")  # or "mlogloss"
    fit_kwargs = dict(X=X_train, y=y_train, eval_set=[(X_valid, y_valid)], verbose=True)

    if "callbacks" in inspect.signature(clf.fit).parameters:
        from xgboost.callback import EarlyStopping
        fit_kwargs["callbacks"] = [EarlyStopping(rounds=50, save_best=True)]
    else:
        fit_kwargs["early_stopping_rounds"] = 50

    clf.fit(**fit_kwargs)

Notes:
- If you were using other advanced callbacks (e.g., LearningRateScheduler), those require either upgrading xgboost or switching to the native xgboost.train API (which has supported callbacks for a long time).
- Quick check: print(xgboost.__version__). If it’s < 1.6, XGBClassifier.fit won’t accept callbacks.

TypeError: LGBMClassifier.fit() got an unexpected keyword argument 'verbose'

You’re hitting two separate issues:

1) LightGBM error (this is what stops your run)
- Cause: In LightGBM ≥ 4.0, LGBMClassifier.fit no longer accepts the verbose keyword. Logging/early stopping moved to callbacks.
- Fix: Remove verbose=... from fit(), and use callbacks (and/or the verbosity parameter) instead.

Example patch:
import lightgbm as lgb

callbacks = [
    lgb.early_stopping(stopping_rounds=50),     # if you used early_stopping_rounds=50
    lgb.log_evaluation(period=100),             # replace verbose=100; use period=0 to silence
]

model = lgb.LGBMClassifier(**params)           # params can include "verbosity": -1 to silence
model.fit(
    X_tr, y_tr,
    eval_set=[(X_val, y_val)],
    eval_metric="logloss",
    callbacks=callbacks
)

If you only wanted to silence logs: model.fit(..., callbacks=[lgb.log_evaluation(period=0)]).

2) XGBoost warnings (not fatal, but good to clean up)
- gpu_hist is deprecated since XGBoost 2.0.
  - Change tree_method="gpu_hist" to tree_method="hist" and set device="cuda" to use GPU, or device="cpu" to stay on CPU.
- "Parameters: { 'predictor' } are not used."
  - Remove predictor (or gpu_predictor) from your params; it’s ignored in 2.x.
- “Falling back to prediction using DMatrix due to mismatched devices…”
  - Your booster is on CUDA but your input array is on CPU. Either:
    - Keep everything on CPU: set device="cpu" when creating the model, or
    - Keep GPU and pass GPU arrays for predict/fit:
        import cupy as cp
        X_tr_gpu = cp.asarray(X_tr.values if hasattr(X_tr, "values") else X_tr)
        y_tr_gpu = cp.asarray(y_tr)
        model = xgb.XGBClassifier(tree_method="hist", device="cuda", **params)
        model.fit(X_tr_gpu, y_tr_gpu, eval_set=[(cp.asarray(X_val), cp.asarray(y_val))])

Quick checklist to apply in your code
- LightGBM train_cv_lgbm(): remove verbose=... from model.fit(...); add callbacks as shown above; optionally set "verbosity": -1 in params.
- XGBoost model creation: use tree_method="hist", device="cuda" (or "cpu"); remove predictor from params; make sure prediction input is on the same device as the model (use CuPy for CUDA).

- Pin older library versions that match your current code:
  - pip install "xgboost==1.7.6" "lightgbm==3.3.5"
