You’re running with mixed precision (fp16), and this line is writing a value that fp16 can’t represent:
logits = logits.masked_fill(mask == 0, -1e9)

In fp16, the most negative finite number is about -65504, so trying to write -1e9 overflows and trips the runtime error. Fix it by using a dtype‑safe fill value or by doing the masking/softmax in float32.

Quick, safe fixes (pick one):

- Dtype-aware minimum (smallest finite value representable by the current dtype):
  replace
    logits = logits.masked_fill(mask == 0, -1e9)
  with
    logits = logits.masked_fill(mask == 0, torch.finfo(logits.dtype).min)

- Use -inf (works fine with softmax and won’t overflow in fp16):
  logits = logits.masked_fill(mask == 0, float('-inf'))

- Do the masking and softmax in float32, then cast back (numerically most stable):
  logits32 = logits.float().masked_fill(mask == 0, float('-inf'))
  attn = torch.softmax(logits32, dim=-1)
  attn = attn.to(logits.dtype)

If your mask is not boolean, make it so (to avoid dtype surprises):
mask = mask.to(torch.bool)

Putting it together, your masked_attn_pool could look like:
def masked_attn_pool(self, x, mask):
    # x: (B, L, D), mask: (B, L) with 1/True for valid tokens
    mask = mask.to(torch.bool)
    logits = self.attn(x).squeeze(-1)          # (B, L)
    # do stable masking/softmax in fp32
    logits32 = logits.float().masked_fill(~mask, float('-inf'))
    attn = torch.softmax(logits32, dim=-1)
    attn = attn.to(x.dtype)
    pooled = torch.einsum('bl,bld->bd', attn, x)
    return pooled

      scaler = torch.cuda.amp.GradScaler()

      scaler = torch.amp.GradScaler('cuda')

      with torch.amp.autocast('cuda', dtype=torch.bfloat16):
